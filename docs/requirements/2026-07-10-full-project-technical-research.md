# 全项目技术调研文档需求

## 目标

对 XianYuAssistant 当前主分支的受版本控制源码、配置、资源、部署脚本和既有说明进行静态调研，形成可在仓库内长期维护的中文技术文档。

## 交付物

- `docs/technical-reference/` 下的一套技术文档集。
- `index.md` 作为统一入口，章节文件按主题拆分，避免单个超级文件。
- 文档集正文总行数不少于 3000 行。
- 内容包含架构、依赖、配置、后端分层、接口、数据模型、消息链路、自动回复、自动发货、前端、部署运维、安全风险和逐文件索引。

## 约束

- 仅做静态阅读和文本级核验。
- 不在本地执行 Maven、Flutter、Gradle、前端构建或测试命令。
- 保留用户现有的 `README.md` 未提交修改，不覆盖、不暂存、不提交。
- 文档不得集中写入一个超级文件；章节应保持可独立阅读和维护。
- 所有中文文件按 UTF-8 读取和写入。
- 完成后生成提交信息、提交并推送，允许临时使用非 GPG 签名提交。

## 验收标准

- 文档集 Markdown 总行数大于等于 3000。
- 索引能够链接到全部章节。
- 逐文件索引覆盖 Java、Vue、TypeScript、主要资源和根目录工程文件。
- 接口索引能够识别 Spring MVC 控制器及映射注解。
- 数据章节覆盖 schema 中的表、索引和触发器。
- 静态检查确认不存在误改业务源码和用户的 README 修改。

## 非目标

- 不修改产品功能。
- 不验证外部闲鱼接口的实时可用性。
- 不运行构建、单元测试、集成测试或浏览器测试。
- 不对已有实现进行重构。

## 完成口径

只有在文档已生成、行数达标、索引与文件覆盖通过静态核验、提交已创建且推送成功后，才可表述为完整完成。
